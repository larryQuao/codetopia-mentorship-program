# Reflection - Week 1 Capstone

## What Are the 3 Git States?

The three Git states are the **Working Directory**, the **Staging Area**, and the **Repository**.

- The Working Directory is where I edit files freely.
- The Staging Area is where I select which changes I want to commit using `git add`.
- The Repository is the permanent history where commits are saved using `git commit`.

-----

## Why Is Git Useful?

Git is useful because it tracks every change made to a project over time. If I make a mistake, I can go back to a previous version. It also allows multiple people to work on the same project without overwriting each other’s work. Without Git, managing code changes would be messy and risky.

-----

## Why Do Commit Messages Matter?

Commit messages matter because they explain *why* a change was made. When I look back at a project months later — or when someone else reads my history — the commit messages tell the story of the project. Vague messages like “fix” or “update” make it impossible to understand what actually changed. Good messages like `docs: add setup notes for session 1` are clear and professional.

-----

## What Did I Learn During Week 1?

During Week 1 I learned how to:

- Install and configure Git with my name and email
- Initialise a repository using `git init`
- Understand the difference between the working directory, staging area, and repository
- Stage files with `git add` and commit them with `git commit`
- Write proper commit messages in imperative tense
- Use `git log --oneline` to view commit history
- Use `git diff --staged` to inspect changes before committing
- Fix a bad commit message using `git commit --amend`

The most important thing I learned is that Git is not just a backup tool — it is a way of thinking carefully and intentionally about changes you make to your code.
